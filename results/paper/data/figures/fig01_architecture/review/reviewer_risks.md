# Reviewer Risks — FIG01

Potential reviewer concerns:
- figure is too abstract and does not look connected to the implementation
- figure is too detailed and unreadable
- acronyms appear before being defined
- blocks are named inconsistently with manuscript text
- architecture implies features not actually implemented

Mitigations:
- keep block names consistent with paper text
- keep the main panel conceptual but faithful
- use a small pipeline/dataflow panel for additional clarity
- do not add unsupported functional claims into the figure
